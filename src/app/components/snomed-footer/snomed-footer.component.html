Assertion Group Help

Group Description
common-authoring This group is used for the application of Drools rules to all extensions (including International) within authoring-platform validation.
int-authoring This group is used for the application of International only Drools rules (those that are not applicable to national extensions) within authoring-platform validation.
xx-authoring Prefixed with a 2 letter county code (e.g. NL) - these groups are used for the application of Extension specific Drools rules within authoring-platform validation.
component-centric This SQL assertion group contains rules validating SNOMED components within the given content. This includes relationships, descriptions, refset entries, concepts and more.
release-type This SQL assertion group contains rules validating release integrity within the given content. It is concerned primarily with data continuity, validity of Snapshot and Delta files, and component states (in comparison to prior states).
release-type This SQL assertion group contains rules validating file integrity within the output RF2 files.
*product*Edition These SQL assertion groups are supersets of the above groups, intended for use in edition validation and release operations by the Release Manager.
Other These SQL assertion groups are supersets of the above groups, intended for use in the validation of specific products. Examples include LOINC and GMDN.
