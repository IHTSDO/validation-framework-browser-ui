Assertion Group Help

Group Description
common-authoring This Drools assertion group is used for validation within the Authoring Platform, and applies to all extensions, including International.
int-authoring This Drools assertion group is used for validation within the Authoring Platform, and applies only to International, and not to national extensions.
xx-authoring Prefixed with a 2 letter county code (e.g. NL) - these Drools assertion groups are used for extension specific validation within the Authoring Platform.
component-centric This SQL assertion group validates SNOMED CT components. For example, relationships, descriptions, refset entries and concepts.
release-type This SQL assertion group validates SNOMED CT release integrity. It is concerned primarily with data continuity, validity of Snapshot and Delta files, and component states (in comparison to prior states).
file-centric-type This SQL assertion group validates file integrity within the output RF2 files.
xxEdition Prefixed with a product name, these SQL assertion groups are supersets of the above, intended for use in edition validation and release operations by the Release Manager.
Other These SQL assertion groups are supersets of the above groups, intended for use in the validation of specific products, for example LOINC and GMDN.
